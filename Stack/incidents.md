# Incidents log

Una lГӯnea por incidente. Formato: `YYYY-MM-DD В· proyecto В· sГӯntoma вҶ’ causa вҶ’ fix (commit/PR)`. Si el aprendizaje universal ya estГЎ en CLAUDE.md, dejarlo aquГӯ solo como pointer.

Para incidentes con anГЎlisis largo (>1 lГӯnea de causa), crear nota separada en `knowledge/incidents/<slug>.md` y enlazarla aquГӯ con `[[wikilink]]`.

## Reglas
- **CronolГіgico inverso** (lo mГЎs reciente arriba).
- **Sin secrets**, sin nombres de cliente sensibles. Proyecto = nombre interno (Simarro, TuFacturaIA, agency-portalвҖҰ).
- Si el fix ya estГЎ en CLAUDE.md como anti-patrГіn, marcar con `вҶ’ CLAUDE.md` para evitar duplicar.
- Si pasa de 80 entradas, archivar las >6 meses a `incidents-archive-YYYY.md`.
